Как это работает
  • Строка заголовков ищется автоматически (по столбцу Summary или Название).
  • Понимаются столбцы: Project/Проект, Summary/Название, Issue Type/Тип, Priority/Приоритет, Status/Статус, Assignee/Исполнитель, Reporter/Автор, Team/Команда, Customer/Заказчик, Sprint/Спринт, Created/Создана, Updated/Обновлена, Description/Описание. Остальные пропускаются.
  • Проекты, команды, заказчики, статусы и спринты создаются автоматически, если таких ещё нет.
  • Исполнитель и автор ищутся по имени среди пользователей системы; незнакомые имена остаются пустыми (пользователи не создаются).
  • Типы Jira: Task → Task, Bug → Bug, Story → Feature, Sub-task → Task, Epic → Epic. Неизвестный тип станет Task.
  • Epic Link привязывает задачу к эпику по названию (эпик ищется среди существующих и создаваемых этим же импортом). Привязка возможна только для Feature — иерархия Epic → Feature → Task/Bug не нарушается, остальные попадут в отчёт ниже.

Экспортированный из трекера файл можно импортировать обратно — столбцы совместимы.
